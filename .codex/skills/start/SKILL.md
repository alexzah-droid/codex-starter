# Skill: Start Session

## Что делать

1. **Прочитать SNAPSHOT.md** — текущее состояние проекта:
   ```bash
   Read .codex/SNAPSHOT.md
   ```
   Если SNAPSHOT.md не существует — создать с базовой структурой (статус: инициализация, дата: сейчас).

2. **Прочитать CODEx.md** — контекст проекта:
   ```bash
   Read CODEx.md
   ```

3. **Проверить git status:**
   ```bash
   git status --short
   git log --oneline -5
   ```

4. **Проверить версию фреймворка** (фоново, не блокирует старт):
   ```bash
   grep -m1 'v[0-9]' README.md 2>/dev/null || true
   ```

5. **Создать лог сессии:**
   ```bash
   mkdir -p .codex/logs/sessions
   ```
   Создать файл `.codex/logs/sessions/YYYY-MM-DD_HH-MM.md` с начальным состоянием (timestamp, SNAPSHOT status, git status).

6. **Доложить пользователю** (кратко, 3-5 строк):
   - Статус проекта
   - Что было сделано в прошлой сессии
   - Незавершённые задачи (если есть)
   - Готовность к работе

## Чего НЕ делать

- Не показывать содержимое файлов пользователю
- Не запускать тесты (это делает `/finish`)
- Не коммитить (если есть незакоммиченное — сообщить, не коммитить автоматически)
- Не тратить больше 30 секунд на инициализацию

## После выполнения

Агент готов принимать ТЗ от пользователя.
