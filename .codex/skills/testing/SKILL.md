# Skill: Testing

## Что делать

1. **Определить доступные тесты:**
   - `pytest`, `npm test`, `cargo test`, `go test` и т.д.

2. **Запустить тесты:**
   - предпочитать команду, которая уже используется в проекте
   - если нет явного тест-скрипта — запускать `pytest` и `npm test` по очереди

3. **Зафиксировать результаты:**
   - прошли/упали
   - количество провалов
   - первые важные ошибки или stack trace

4. **Обновить SNAPSHOT.md:**
   - добавить краткий результат тестирования
   - зафиксировать состояние после проверки

## Чего НЕ делать

- Не менять тестовую среду без явного указания
- Не коммитить результаты тестов автоматически
- Не считать тесты пройденными по неявному выводу
