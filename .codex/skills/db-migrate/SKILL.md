# Skill: DB Migrate

## Что делать

1. **Проверить текущую базу:**
   - `.sqlite`, `.db`, `migrations/`
   - конфигурацию database URL

2. **Собрать план миграции:**
   - локальные шаги
   - необходимые резервные копии
   - перевод в staging/production

3. **Запустить миграцию:**
   ```bash
   ./scripts/db-migrate.sh 2>/dev/null || true
   ```
   или соответствующую команду проекта.

4. **Сохранить результат:**
   - статус миграции
   - ошибки
   - рекомендации по следующему шагу

## Чего НЕ делать

- Не мигрировать данные без резервной копии
- Не коммитить секреты подключения
- Не выполнять необратимые операции без подтверждения
