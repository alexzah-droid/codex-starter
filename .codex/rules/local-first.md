# Правило: Local-first

## Принцип

Работай с локальными ресурсами прежде, чем переходить в облако. Это снижает риски и ускоряет итерации.

## Что делать сначала
- Запускать код локально
- Использовать локальные базы данных и файлы
- Проверять изменения без внешних зависимостей
- Убедиться, что поведение воспроизводимо локально

## Когда переходить в облако
- Когда локальное состояние стабилизировано
- Когда необходимо протестировать интеграцию
- Когда проект готов к staging/production

## Миграции

- Для миграции хранить схему и данные отдельно
- Прописать шаги миграции в `scripts/`
- Не выполнять миграцию без резервной копии

## Для Codex starter

- `.codex/SNAPSHOT.md` остаётся локально в shared/public режиме
- `.codex/settings.local.json` используется для локальных переменных и секретов
- Хуки и логирование работают локально, без автоматической отправки данных
