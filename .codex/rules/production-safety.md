# Правило: Production Safety

## Production deploy только с подтверждением

Любая операция, связанная с production deploy, release или публикацией, требует явного подтверждения пользователя.

### Когда спрашивать пользователя
- `deploy` в production
- `publish` релиза
- изменение инфраструктуры или облачных ресурсов
- миграция данных в production

### Что можно делать без подтверждения
- локальные разработки
- тесты и проверка кода
- подготовка артефактов
- работа с документацией и rules

### Как действовать при неопределённости
1. Определи, что считается production для этого проекта
2. Если действие затрагивает live system — запроси подтверждение
3. Если нет возможности подтвердить — остановись и опиши риск

## Безопасность данных

- Не публикуй секреты
- Не добавляй credentials в git
- Если требуется подключение к API — используй environment-переменные
- Не выполняй destructive команды без явного разрешения
